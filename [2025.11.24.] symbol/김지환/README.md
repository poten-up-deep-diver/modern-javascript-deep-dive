# 심벌(Symbol)

# 🤨Symdol이란

심벌(Symbol)은 ECMAScript 2015(ES6)에서 도입된 자바스크립트의 **7번째 데이터 타입**으로, 변경 불가능(immutable)하고 유일무이(unique)한 원시 값(Primitive Value)이다.

## Symbol의 생성

심볼은 2가지 방법으로 생성할수 있다.

- Symbol()
- Symbol.for()

## Symbol(), Symbol.for()가 뭘까

- **`Symbol()`**: 무조건 새로운 심벌을 만든다.
- **`Symbol.for()`**: 이미 있는 건 공유하고, 없으면 만든다.

## Symbol() 예시

```jsx
const a = Symbol("kim");
const b = Symbol("kim");

console.log(a === b); // false
// "kim"은 같지만, 메모리상에서는 서로 다르기 때문이다.
```

## Symbol.for() 예시

```jsx
const c = Symbol.for("kim");
const d = Symbol.for("kim");

console.log(c === d); // true
// 레지스트리에 'kim'라는 이름으로 등록된 심벌을 공유해서 가져왔기 때문이다.
```

# 🤨심벌 장단점

## 심벌 장점

- 고유성(Unique) 보장
  → 객체 키로 사용할 때 다른 코드와 충돌할 걱정이 없다.
- 불변성(Immutable)
  → 생성된 심벌 값은 변하지 않으며, 안전하게 키나 식별자로 사용할 수 있다.
- 충돌 없는 확장 가능하다.
  → 라이브러리나 다른 코드와 이름이 겹치지 않아 안전하게 객체를 확장할 수 있다.

## 심벌 단점

- 문자열처럼 자동 변환 안 된다.
  → 문자열 + Symbol 같이 쓰면 오류 남 (`TypeError`).
- 디버깅 불편
  → 콘솔에 찍어도 어떤 Symbol인지 직관적으로 보기 어렵고, 이름이 안 뜰 때가 있다.
- 문자열 기반 JSON에 포함 안 된다.
  → `JSON.stringify()` 사용하면 Symbol 속성은 아예 사라진다.

# 🤨심벌 왜쓰고, 어디서 쓰는

### 왜 쓰냐?

1. **프로퍼티 이름 충돌 방지**
   - 라이브러리, 코드가 커져도 키가 절대 겹치지 않는다.
2. **실수로 접근/수정되는 것 방지**
   - `for...in`, `Object.keys` 등에서 안 보임 → 은근히 숨겨진다.
3. **유일한 상수 정의 가능**
   - 값이 절대 중복되지 않아 “의미 있는 상태값” 만들 때 좋다.

## 어디서 쓰냐?

- **객체의 숨겨진(중복되지 않는) 키로 사용**
  → 다른 키와 절대 충돌하지 않아서 라이브러리·코드 내부 전용 프로퍼티 만들 때 사용한다.
- **React 기반 서비스들(대부분의 현대 웹사이트)**
  → React 내부에서 컴포넌트 판별, 요소 타입 식별 등에 Symbol을 사용한다.

# 💯결론

- 심벌(Symbol)은 절대 충돌하지 않는 고유한 값을 만들기 위해 사용되는 ES6의 새로운 원시 타입이다.
- 심벌(Symbol)은 충돌 없이 안전하게 객체 키를 만들 수 있지만, 문자열 변환·디버깅·JSON 처리에서는 불편함이 있는 타입이다.
- 심벌은 충돌 없는 고유 키가 필요할 때, 라이브러리나 React에서 요소 식별용으로 사용된다.
