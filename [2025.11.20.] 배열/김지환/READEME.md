## 🤔 ES6 이전과 이후 함수 구분

### 1. ES6 이전 함수 특징

- 모든 함수가 호출 가능(callable) 하면서 동시에 생성 가능(constructor) 했다.
- 객체에 바인딩된 함수도 메서드처럼 호출 가능하지만, 동시에 생성자로도 사용 가능 → 혼란 발생한다.
- 콜백 함수도 생성자로 동작해 불필요한 프로토타입이 생기고 성능 저하 발생한다.
- 즉, 모든 함수가 동일하게 동작했기 때문에 실수와 비효율이 많았다.

---

### 2. ES6 이후 함수 구분

ES6에서는 함수의 목적에 따라 명확히 구분되도록 개선된다.

| 함수 종류   | 생성자(constructor) | 프로토타입(prototype) | super 사용 | arguments |
| ----------- | ------------------- | --------------------- | ---------- | --------- |
| 일반 함수   | O                   | X                     | X          | O         |
| 메서드      | X                   | X                     | O          | O         |
| 화살표 함수 | X                   | X                     | X          | X         |

- **일반 함수**: 생성자 호출 가능, 기존 JS 함수와 동일하다.
- **메서드**: 객체 리터럴 축약 표현, 생성자 불가, `super` 사용 가능하다
- **화살표 함수**: 화살표 함수는 짧게 쓰고, 생성자는 안 되고, `this`, `arguments`, `super`는 주변 환경에서 그대로 가져와서 사용한다.

---

### 3. 핵심 요지

**ES6 이후 함수는 목적에 따라 기능과 제약이 구분되어, 혼란 최소화와 성능 개선이 가능하다.**

## 🤔 배열

### 배열 정의

- 배열(Array)은 여러 값을 순서대로 담는 동적 리스트 구조이며, 인덱스로 요소에 접근하고 다양한 메서드로 조작할 수 있다.

---

### 배열 특징

1. **길이(length)**: 마지막 인덱스 기준으로 계산되었다.
2. **객체 특성**: 속성 추가 가능하다.
3. **동적 크기**: 요소 추가/삭제 시 자동으로 크기 조절됨 (`push`, `pop` 등)

---

## **🤔**배열 고차 함수란?

- 배열 메서드 중에서 함수(콜백)를 인자로 받는 것을 말한다.
- 쉽게 말해 배열 요소를 다룰 때 콜백 함수를 사용하는 메서드가 배열 고차 함수이다.

### 배열 고차 함수 종류

| 메서드        | 설명                                           | 반환값       |
| ------------- | ---------------------------------------------- | ------------ |
| `forEach()`   | 배열 요소를 순회하며 콜백 실행                 | `undefined`  |
| `map()`       | 배열 요소를 변형 새로운 배열을 만듬            | 새 배열      |
| `filter()`    | 조건에 맞는 요소만 추출                        | 새 배열      |
| `reduce()`    | 배열의 모든 요소를 하나의 값으로 합치거나 축약 | 단일 값      |
| `some()`      | 조건을 만족하는 요소가 하나라도 있는지 확인    | `true/false` |
| `every()`     | 모든 요소가 조건을 만족하는지 확인             | `true/false` |
| `find()`      | 조건에 맞는 첫 요소 반환                       | 요소 값      |
| `findIndex()` | 조건에 맞는 첫 요소 인덱스 반환                | 숫자         |

---

### 배열 단점

1. **큰 배열은 메모리를 많이 사용한다.**
   - 요소가 많아질수록 메모리 부담도 커진다.
2. **중첩 배열은 구조가 복잡해진다.**
   - 깊게 들어갈수록 접근과 관리가 어려워진다.
3. **여러 타입이 섞이면 관리가 힘들다.**
   - 숫자, 문자열, 객체 등이 섞여 있으면 데이터 일관성이 떨어져 헷갈릴 수 있다.

## 🤔 정규표현식(RegExp)

### 1. 개념

정규표현식은 **특정 패턴을 가진 문자열을 찾거나 치환**할 때 사용하는 도구이다.

---

### 2. 패턴 종류

| 분류        | 설명                   | 예시                          |
| ----------- | ---------------------- | ----------------------------- |
| 리터럴 문자 | 문자 그대로 찾기       | `abc`                         |
| 문자 집합   | 여러 문자 중 하나 선택 | `[abc]`                       |
| 범위 지정   | 문자 범위 지정         | `[a-z]`, `[0-9]`              |
| 메타문자    | 특수 의미 기호         | `.`, `\d`, `\w`               |
| 앵커        | 문자열 경계            | `^`, `$`                      |
| 반복        | 패턴 반복 지정         | `*`, `+`, `?`, `{n}`, `{n,m}` |
| 그룹        | 여러 패턴 묶음         | `(abc)`, `(cat                |
| 백레퍼런스  | 그룹 재참조            | `\1`, `\2`                    |
| 플래그      | 검색 방식 설정         | `g`, `i`, `m`, `s`, `u`, `y`  |

---

### 3. 주요 메서드

- `exec()`: 패턴 일치 결과 반환한다.
- `test()`: 일치 여부(true/false) 반환한다.

---

### 4. 플래그

- `i`: 대소문자 무시
- `g`: 전체 문자열 반복 검색
- `m`: 여러 줄 검색

---

## 5. 단점

1. **작성 어려움**
   - 복잡한 패턴일수록 만들기 어렵다.
2. **가독성 낮음**
   - 한눈에 무슨 의미인지 파악하기 힘들다.
3. **디버깅 어려움**
   - 오류가 나면 어디가 문제인지 찾기 어렵다.

---

## 🤔 String

### 1. 정의

**String**은 여러 문자를 이어 만든 **텍스트 데이터**이다.

대부분 언어에서 문자열은 **불변(immutable)**이라, 수정하려면 새 문자열을 만들어야 한다.

---

# **🤔**String 메소드

## 1. InCludes

- includes 메소드 : 배열이나 문자열 안에 특정 값이 있는지 확인해 `true` 또는 `false`를 반환하는 메서드
- 왜 쓸까? : 값 존재 여부를 간단하고 직관적으로 확인하기 위해 사용한다.
- 어디서 쓸까? : 조건문, 배열 필터링, 문자열 검색 등 값 포함 여부가 필요한 곳에서 쓴다.

## 2. slice

- slice 메소드 : 배열이나 문자열의 일부를 잘라 새 배열이나 새 문자열로 반환하는 메서드이다.
- 왜 쓸까? \*\*\*\*: 필요한 부분만 추출하거나, 원본을 유지하면서 부분 복사할 때 사용한다.
- 어디서 쓸까? \*\*\*\*: 배열 일부 선택, 문자열 일부 추출, 페이징 처리, 데이터 복사 등 쓰안다

## 3. trim

- trim 메소드 : 대상 문자열 앞뒤에 여백이 있을 경우 여백을 지워준다.
- 왜 쓸까? \*\*\*\*: 사람의 실수를 막고, 입력 오류도 막을수있다.
- 어디서 쓸까? : 로그인이나 검색에서 사용된다. 쓰는이유는 사용자가 실수로 생긴 공백 때문에 생기는 오류를 막아준다

---

### String 단점

1. **불변성**
   - 문자열은 수정이 안 되어서, 바꾸려면 항상 새 문자열을 만들어야 한다.
2. **성능·메모리 부담**
   - 문자열을 반복해서 다루거나 많이 이어 붙이면 \*\*\*\*속도가 느려지고 메모리 사용도 늘어난다.
3. **멀티바이트 문자 처리 어려움**
   - 한글, 이모지처럼 글자 크기가 다른 문자들은 길이 계산이나 잘라내기에서 문제가 생길 수 있어 주의가 필요하다.
