# 클래스와 super키워드

## **✋🏻** 들어가기전

클래스를 설명하기 전에 **“클래스는 프로토타입의 문법적 설탕인가?”**를 설명하고 시작할게

### 🤨 문법적 설탕은 므야?

일단 문법적 설탕은 **“기존 기능을 더 편하게 짧게 쓰게해주는 문법 편의 기능”**

### 🤨 **클래스는 프로토타입의 문법적 설탕인가**

그래서 왜 클래스는 프로토타입의 문법적 설탕인가 이유는 클래스는 **내부적으로 프로토타입을 그**

**대로 사용하면서** 그걸 더 읽기 좋고 쓰기 쉽게 만들어 놓은 **“편한 문법”**이라서

문법적 설탕이라고 부른다.

## **🤔** 클래스란

클래스는 여러 객체를 쉽게 만들고 관리하기 위해 만들어진 **‘공통 설계도’**이다.

### 🤨 \*\*\*\*클래스 특징이 므야?

1.  클래스도 **“값”**처럼 사용할수있다.(**일급 객체**)

        클래스를 **변수**에 담고, **함수**에 넘기고, **반환**할 수도 있어

1.  상속(Inheritance))이 가능하다.

        `extends` 를 사용하면 부모 클래스 기능을 자식 클래스를

        그대로 물려받을 수 있다.

1.  객체를 만들기 위한 ‘설계도’

    클래스는 여러 객체를 찍어낼 수 있는 틀(템플릿) 역활을 한다.

### 🤨클래스 장단점은 므야?

### 💯 장점

1. 재사용성이 다.
2. 코드 구조가 깔끔해진다.
3. 일급 객체처럼 활용이 가능하다.

### 🚫 단점

1. 함수보다 복잡할 떄가 있다.
2. this떄문에 햇갈릴 수 있다.

   (클래스 안에서 this가 가리키는 게 뭔지 햇갈릴 떄가 있다.)

3. 메모리 낭비 가능하다.

   (생성자 안에 함수를 넣으면 객체마 새로 만들어짐)

## **🤔** super 키워드란

`super` 키워드는 함수처럼 호출할 수도 있고 this와 같이 식별자처럼 참조할 수 있는

특수한 키워드다

### **🤔super 쓰는 이유는**

상속받은 **부모 클래스**의 속성이나 메서드를 그대로 사용하거나, 조금 바꿔서 쓰고싶을떄

사용하는 키워드이다.

## **🧑🏻‍⚖️ 결론**

1. ES6부터 도입된 클래스는 내부적으로는 프로토타입 기반과 같지만, 개발자가 보기 쉽고 쓰기 편하도록 만든 문법적 설탕이다.

2. 클래스는 객체를 효율적으로 만들고 관리할 수 있는 설계도로, 재사용성과 가독성에 유리하지만 this와 구조 복잡성에 주의가 필요하다.

3. `super`는 자식 클래스에서 부모 클래스의 속성과 메서드를 재사용하거나 확장할 때 사용하는 키워드이다.
